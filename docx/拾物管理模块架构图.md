# 拾物管理模块架构图

## 架构概览

```
前端 → FoundItemController → FoundItemService → FoundItemServiceImpl → 数据访问层(MyBatis Plus) → MySQL
                                         ↓
                                      DTO层(FoundItem)
```

## 各层详细说明

### 1. Controller 层 (FoundItemController)

- **位置**: `/backend/src/main/java/com/campus/lostfound/found/FoundItemController.java`
- **职责**: 处理 HTTP 请求，提供 RESTful API 接口
- **主要接口**:
  - `GET /api/found-items`: 分页查询拾物信息列表，支持关键词搜索
  - `GET /api/found-items/{id}`: 获取特定拾物信息详情
  - `POST /api/found-items`: 发布新的拾物信息
  - `PUT /api/found-items/{id}`: 更新拾物信息
  - `DELETE /api/found-items/{id}`: 删除拾物信息
- **异常处理**: 包含参数验证和业务异常处理
- **返回格式**: 统一使用 ApiResponse 格式

### 2. Service 接口层 (FoundItemService)

- **位置**: `/backend/src/main/java/com/campus/lostfound/found/service/FoundItemService.java`
- **职责**: 定义拾物管理业务逻辑接口
- **主要方法**:
  - `getFoundItemsPage`: 分页获取拾物信息
  - `getFoundItemById`: 根据 ID 获取拾物详情
  - `createFoundItem`: 创建拾物信息
  - `updateFoundItem`: 更新拾物信息
  - `deleteFoundItem`: 删除拾物信息

### 3. Service 实现层 (FoundItemServiceImpl)

- **位置**: `/backend/src/main/java/com/campus/lostfound/found/service/impl/FoundItemServiceImpl.java`
- **职责**: 实现具体的业务逻辑
- **核心逻辑**:
  - 使用 MyBatis Plus 进行数据库操作
  - 实现分页查询功能
  - 数据实体与 DTO 之间的转换
  - 参数验证（验证物品名称、描述、地点等不能为空）
  - 设置默认值（如 scene="found", stage="active", auditStatus="PENDING"等）
  - 生成业务 ID
  - 获取当前用户 ID 作为物品拥有者
- **依赖注入**: 注入 ItemMapper 进行数据访问和 SecurityUserUtils 获取当前用户

### 4. DTO 层 (FoundItem)

- **位置**: `/backend/src/main/java/com/campus/lostfound/found/dto/FoundItem.java`
- **职责**: 数据传输对象，用于在不同层级间传递数据
- **主要属性**:
  - `id`: 拾物 ID
  - `itemName`: 物品名称
  - `description`: 物品描述
  - `location`: 拾取地点
  - `contactName`: 联系人姓名
  - `status`: 状态
  - `userId`: 用户 ID
  - `createTime`: 创建时间

### 5. 数据访问层 (ItemMapper)

- **位置**: `/backend/src/main/java/com/campus/lostfound/mapper/ItemMapper.java`
- **职责**: 数据库操作接口
- **技术**: 使用 MyBatis Plus 框架
- **主要方法**: 基于通用 Mapper 的增删改查操作

### 6. 实体层 (Item)

- **位置**: `/backend/src/main/java/com/campus/lostfound/domain/entity/Item.java`
- **职责**: 对应数据库表的实体类
- **映射**: 与 MySQL 数据库中的 items 表对应

### 7. 工具类

- **SecurityUserUtils**: 用于获取当前用户身份信息
- **BeanUtils**: Spring 框架提供的对象属性复制工具
