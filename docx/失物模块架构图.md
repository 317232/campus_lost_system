### 各层详细说明

### 示例 ： 失物模块

#### 1. Controller 层 (LostItemController)

**文件位置**: `lost/LostItemController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |------|------|------| | GET | `/api/lost-items` | 分页查询失物列表，支持关键词搜索 | | GET | `/api/lost-items/{id}` | 获取指定失物详情 | | POST | `/api/lost-items` | 创建新的失物信息 | | PUT | `/api/lost-items/{id}` | 更新失物信息 | | DELETE | `/api/lost-items/{id}` | 删除失物信息 |

------

#### 2. Service 接口层 (LostItemService)

**文件位置**: `lost/service/LostItemService.java`

定义了失物模块的业务接口，封装业务流程：

```
public interface LostItemService {
    Page<LostDTO.LostItemResp> getLostItemsPage(Integer pageNum, Integer pageSize, String keyword);
    LostDTO.LostItemResp getLostItemById(Long id);
    void createLostItem(LostDTO.CreateLostReq request);
    boolean updateLostItem(LostDTO.UpdateLostReq request);
    boolean deleteLostItem(Long id);
}
```

------

#### 3. Service 实现层 (LostItemServiceImpl)

**文件位置**: `lost/service/Impl/LostItemServiceImpl.java`

实现具体的业务逻辑：

| 业务方法 | 核心逻辑 | |---------|---------| | **getLostItemsPage()** | ① 创建分页对象 ② 构建查询条件，scene="lost" ③ 关键词搜索 itemName/description ④ 分页查询 ⑤ 实体转换 DTO | | **getLostItemById()** | ① 按ID查询 ② 验证 scene="lost" ③ 转换 DTO 返回 | | **createLostItem()** | ① DTO 转实体 ② 设置 scene="lost", stage="active", status="PUBLISHED" ③ 插入数据库 | | **updateLostItem()** | ① 验证ID ② 查询并验证 scene="lost" ③ 属性拷贝 ④ 更新数据库 | | **deleteLostItem()** | ① 查询并验证 scene="lost" ② 按ID删除 |

------

#### 4. DTO 层 (LostDTO)

**文件位置**: `lost/dto/LostDTO.java`

数据传输对象，用于请求和响应：

| 类名 | 用途 | |------|------| | **CreateLostReq** | 创建失物请求: itemName, description, categoryId, location, timeLabel, contactPhone, images | | **UpdateLostReq** | 更新失物请求: id, itemName, description, categoryId, location, timeLabel, contactPhone, images | | **LostItemResp** | 失物响应: id, bizId, itemName, description, categoryId, categoryName, location, timeLabel, contactPhone, images, status, userId, userName, createTime | | **LostQueryReq** | 查询请求: pageNum, pageSize, keyword, status, categoryId |

------

#### 5. Mapper 层 (ItemMapper)

**文件位置**: `mapper/ItemMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

- `selectPage()` - 分页查询
- `selectById()` - 按ID查询单条记录
- `insert()` - 插入记录
- `updateById()` - 按ID更新
- `deleteById()` - 按ID删除

------

#### 6. 实体层 (Item)

**文件位置**: `domain/entity/Item.java`

数据库表对应的实体类：

| 字段 | 说明 | |------|------| | id | 主键ID | | bizId | 业务ID | | scene | 场景(lost/found) | | stage | 阶段 | | status | 状态 | | itemName | 物品名称 | | description | 物品描述 | | location | 地点 | | timeLabel | 时间标签 | | contactPhone | 联系电话 | | images | 图片 | | createTime | 创建时间 |
