### 各层详细说明

### 示例 ： 管理员模块

#### 1. Controller 层 (AdminController)

**文件位置**: `admin/AdminController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 功能 | 方法 | 路径 | 说明 | |----|----|----|----| |物品审核|POST|/api/admin/items/audit|审核物品| |认领审核|POST|/api/admin/claims/audit|审核认领申请| |公告列表|GET|/api/admin/notices|获取公告列表| |创建公告|POST|/api/admin/notices|创建新公告| |更新公告|PUT|/api/admin/notices/{id}|更新公告| |删除公告|DELETE|/api/admin/notices/{id}|删除公告| |分类列表|GET|/api/admin/categories|获取分类列表| |创建分类|POST|/api/admin/categories|创建分类| |更新分类|PUT|/api/admin/categories/{id}|更新分类| |删除分类|DELETE|/api/admin/categories/{id}|删除分类| |举报列表|GET|/api/admin/reports|获取举报列表| |处理举报|PUT|/api/admin/reports/{id}|处理举报| |删除举报|DELETE|/api/admin/reports/{id}|删除举报| |数据统计|GET|/api/admin/stats/dashboard|获取统计数据| |用户列表|GET|/api/admin/users|获取用户列表| |创建用户|POST|/api/admin/users|创建用户| |更新用户|PUT|/api/admin/users/{id}|更新用户| |删除用户|DELETE|/api/admin/users/{id}|删除用户| |更新用户状态|PUT|/api/admin/users/{id}/status|启用/禁用用户|

------

#### 2. Service 接口层 (AdminService)

**文件位置**: `admin/service/AdminService.java`

定义了管理员模块的业务接口：

```
public interface AdminService {
    // 审核
    void auditItem(AuditItemReq request);
    void auditClaim(AuditClaimReq request);
    
    // 公告CRUD
    List<NoticeResp> listNotices();
    Long createAnnouncement(AnnouncementCreateReq request);
    boolean updateAnnouncement(AnnouncementUpdateReq request);
    boolean deleteAnnouncement(Long announcementId);
    
    // 分类CRUD
    List<CategoryResp> listCategories();
    Long createCategory(CategoryUpsertReq request);
    boolean updateCategory(CategoryUpsertReq request);
    boolean deleteCategory(Long categoryId);
    
    // 数据统计
    DashboardResp dashboard();
    
    // 举报CRUD
    List<ReportResp> listReports();
    boolean handleReport(HandleReportReq request);
    boolean deleteReport(Long reportId);
    
    // 用户管理
    List<UserListResp> listUsers();
    Long createUser(CreateUserReq request);
    boolean updateUser(Long userId, UpdateUserReq request);
    boolean deleteUser(Long userId);
    void updateUserStatus(Long userId, String status);
}
```

------

#### 3. Service 实现层 (AdminServiceImpl)

**文件位置**: `admin/service/impl/AdminServiceImpl.java`

实现具体的业务逻辑：

| 业务方法 | 核心逻辑 | |---------|---------| |**auditItem()**|① 验证物品存在 ② 更新audit_status ③ 通过则status=PUBLISHED，否则REJECTED ④ 记录审核日志到item_dault表(@Transactional)| |**auditClaim()**|① 验证认领申请存在 ② 更新status ③ 通过则更新物品status=CLAIMED, stage=closed(@Transactional)| |**公告CRUD**|list:查询所有，倒序；create:插入，返回ID；update:更新；delete:删除| |**分类CRUD**|list:按sort_order正序；create:生成bizId=CAT+UUID；update:更新；delete:删除| |**举报CRUD**|list:倒序；handle:更新status、handle_remark、handled_at、handled_by；delete:删除| |**dashboard()**|统计:失物数、招领数、总认领数、通过认领数、成功率| |**用户CRUD**|list:全量用户；create:验证学号唯一性，密码BCrypt加密；update:更新信息；delete:删除；updateStatus:ACTIVE/INACTIVE(@Transactional)|

#### 4. DTO 层 (AdminDTO)

**文件位置**: `admin/dto/AdminDTO.java`

数据传输对象：

| 类 | 用途 | |----|----| |AuditItemReq|物品审核请求: itemId, auditStatus, auditRemark| |AuditClaimReq|认领审核请求: claimId, auditStatus, auditRemark| |AnnouncementCreateReq|创建公告: title, content, published| |AnnouncementUpdateReq|更新公告: 继承CreateReq + id| |CategoryUpsertReq|分类请求: id, name, sortOrder, status| |HandleReportReq|处理举报请求: reportId, status, handleRemark| |DashboardResp|统计响应: lostCount, foundCount, approvedClaims, totalClaims, claimSuccessRate| |NoticeResp|公告响应: id, title, content, published, publishedAt, createTime| |CategoryResp|分类响应: id, bizId, name, sortOrder, status, createTime| |ReportResp|举报响应: id, bizId, reporterId, targetType, targetId, reason, detail, status, handleRemark, handledAt, createTime| |UserListResp|用户响应: id, studentNo, name, phone, email, major, avatarUrl, status| |CreateUserReq|创建用户请求: studentNo, name, password, phone, email, major| |UpdateUserReq|更新用户请求: name, phone, email, major| |UpdateUserStatusReq|更新用户状态请求: status|
