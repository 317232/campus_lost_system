### 各层详细说明

### 示例 ： 分类模块

#### 1. Controller 层 (CategoryController)

**文件位置**: `catalog/CategoryController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |----|----|----| |GET|/api/categories|获取所有分类列表| |GET|/api/categories/{id}|获取分类详情| |POST|/api/categories|创建分类| |PUT|/api/categories/{id}|更新分类| |DELETE|/api/categories/{id}|删除分类|

------

#### 2. Service 接口层 (CategoryService)

**文件位置**: `catalog/service/CategoryService.java`

定义了分类模块的业务接口：

```
public interface CategoryService {
    List&lt;CategoryResp&gt; getAllCategories();
    CategoryResp getCategoryById(Long id);
    void createCategory(CreateCategoryReq request);
    boolean updateCategory(UpdateCategoryReq request);
    boolean deleteCategory(Long id);
}
```

------

#### 3. DTO 层

数据传输对象：

| 类 | 用途 | |----|----| |CreateCategoryReq|创建分类请求: name, sortOrder, status| |UpdateCategoryReq|更新分类请求: id, name, sortOrder, status| |CategoryResp|分类响应: id, bizId, name, sortOrder, status, createTime|

------

#### 4. Mapper 层 (CategoryMapper)

**文件位置**: `mapper/CategoryMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

- selectList() - 查询列表
- selectById() - 按ID查询
- insert() - 插入
- updateById() - 按ID更新
- deleteById() - 按ID删除

------

#### 5. 实体层 (Category)

数据库表对应的实体类，字段包括：
- id (主键)
- biz_id (业务ID: CAT+UUID)
- name (分类名称)
- sort_order (排序)
- status (状态: ENABLED/DISABLED)
- create_time (创建时间)
