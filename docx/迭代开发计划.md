## 一、问题优先级分析

根据需求分析报告，系统存在以下级别的待解决问题：

### 🔴 P0 优先级（阻塞性问题，必须优先处理）

| 编号  | 问题                    | 影响                                                                                                 | 建议解决方案                                   |
| ----- | ----------------------- | ---------------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| P0-01 | **后端控制器架构偏差**  | 数据库设计为统一 `items` 表，但后端存在 `LostItemController` 和 `FoundItemController` 两套独立控制器 | 统一为 `ItemController`，通过 `scene` 参数区分 |
| P0-02 | **文件上传接口缺失**    | 无法完成图片附件功能，发布信息中的图片无法上传                                                       | 实现 `FileUploadController`                    |
| P0-03 | **前端 API 路径不匹配** | 前端调用 `/api/lost-items` 和 `/api/found-items`，与统一 API 设计不符                                | 统一为 `/api/items`                            |

### 🟡 P1 优先级（重要功能，需尽快实现）

| 编号  | 问题                     | 影响                                          | 建议解决方案                        |
| ----- | ------------------------ | --------------------------------------------- | ----------------------------------- |
| P1-01 | **物品搜索筛选不完整**   | API 缺少 category/zone/sort 参数              | 完善 `ItemController` 的筛选参数    |

## 三、详细任务清单

### 任务 P0-01：统一物品 API

**问题描述**：
数据库设计为统一 `items` 表，但后端同时存在 `LostItemController` 和 `FoundItemController` 两套独立控制器，与 init.sql 架构偏差。

**解决方案**：

1. 新建 `ItemController`（`/api/items`）
2. 新建 `ItemService` 接口
3. 新建 `ItemServiceImpl` 实现
4. 通过 `scene` 参数（lost/found）区分失物和招领
5. 更新前端 API 调用

**文件清单**：

```
backend/src/main/java/com/campus/lostfound/
├── items/
│   ├── ItemController.java (新建)
│   ├── dto/
│   │   └── ItemDTO.java (新建)
│   └── service/
│       ├── ItemService.java (新建)
│       └── impl/
│           └── ItemServiceImpl.java (新建)
```

**API 设计**：
| 方法 | 路径 | 功能 |
| ---- | ---- | ---- |
| GET | `/api/items` | 物品列表（支持 scene/keyword/category/zone 参数） |
| GET | `/api/items/{id}` | 物品详情 |
| POST | `/api/items` | 创建物品（scene 参数区分失物/招领） |
| PUT | `/api/items/{id}` | 更新物品 |
| DELETE | `/api/items/{id}` | 删除物品 |
| GET | `/api/items/me` | 我的发布记录 |

---

### 任务 P0-02：文件上传接口

**问题描述**：
init.sql 中 `item_attachments` 表设计完整，但后端无 `FileUploadController`，无法完成图片附件功能。

**解决方案**：

1. 创建 `FileUploadController`
2. 实现单文件和多文件上传
3. 配置文件存储路径
4. 返回文件访问路径

**文件清单**：

```
backend/src/main/java/com/campus/lostfound/
├── file/
│   ├── FileUploadController.java (新建)
│   └── FileService.java (新建)
```

**API 设计**：
| 方法 | 路径 | 功能 |
| ---- | ---- | ---- |
| POST | `/api/files/upload` | 单文件上传 |
| POST | `/api/files/uploads` | 多文件上传 |
| GET | `/api/files/{filename}` | 文件访问 |

---

### 任务 P1-03：认领草稿流程

**问题描述**：
`claim_drafts` 表和 `ClaimDraft.java` 实体存在，但 API 流程（draft → formal）未完整实现。

**解决方案**：

1. 实现 `POST /api/claims/draft` 关键词验证接口
2. 生成草稿令牌（24 小时有效）
3. 实现正式认领申请接口（携带 draft_id）
4. 完善状态流转逻辑

**API 设计**：
| 方法 | 路径 | 功能 |
| ---- | ---- | ---- |
| POST | `/api/claims/draft` | 提交认领草稿（关键词验证） |
| POST | `/api/claims` | 正式认领申请（携带 draft_id） |
