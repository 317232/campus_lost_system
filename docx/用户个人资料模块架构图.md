# 用户个人资料模块架构图

## 架构概览

```
前端 → ProfileController → ProfileService → ProfileServiceImpl → 数据访问层(MyBatis Plus) → MySQL
                                        ↓
                                     DTO层(UserDTO, UserProfileResponse)
```

## 各层详细说明

### 1. Controller 层 (ProfileController)

- **位置**: `/backend/src/main/java/com/campus/lostfound/user/ProfileController.java`
- **职责**: 处理 HTTP 请求，提供 RESTful API 接口
- **主要接口**:
  - `GET /api/users/me`: 获取当前用户资料
  - `PUT /api/users/me`: 更新当前用户资料
  - `GET /api/users/me/lost-items`: 获取当前用户发布的失物信息
  - `GET /api/users/me/found-items`: 获取当前用户发布的拾物信息
  - `GET /api/users/statistics`: 获取用户统计数据
  - `GET /api/users/{userId}/lost-items`: 获取指定用户发布的失物信息
  - `GET /api/users/{userId}/found-items`: 获取指定用户发布的拾物信息
- **异常处理**: 包含参数验证和业务异常处理
- **返回格式**: 统一使用 ApiResponse 格式

### 2. Service 接口层 (ProfileService)

- **位置**: `/backend/src/main/java/com/campus/lostfound/user/service/ProfileService.java`
- **职责**: 定义用户资料业务逻辑接口
- **主要方法**:
  - `getCurrentProfile`: 获取当前用户资料
  - `getCurrentUserProfile`: 获取当前用户详细资料
  - `updateCurrentProfile`: 更新当前用户资料
  - `getUserLostItems`: 获取用户发布的失物信息
  - `getUserFoundItems`: 获取用户发布的拾物信息
  - `getAdminStatistics`: 获取用户统计数据

### 3. Service 实现层 (ProfileServiceImpl)

- **位置**: `/backend/src/main/java/com/campus/lostfound/user/service/impl/ProfileServiceImpl.java`
- **职责**: 实现具体的业务逻辑
- **核心逻辑**:
  - 获取当前用户信息
  - 更新用户资料（姓名、电话、邮箱、头像等）
  - 获取用户的失物/拾物信息
  - 计算用户统计数据（发布数量、审核状态、通过率等）
  - 获取用户权限和角色信息
  - 用户数据与 DTO 之间的转换
- **依赖注入**: 注入 UserMapper、ItemMapper、RoleMapper、PermissionMapper 进行数据访问和 SecurityUserUtils 获取当前用户

### 4. DTO 层 (UserDTO, UserProfileResponse)

- **位置**: `/backend/src/main/java/com/campus/lostfound/user/dto/`
- **职责**: 数据传输对象，用于在不同层级间传递数据
- **主要类型**:
  - `UserResp`: 用户基本信息响应
  - `UserItemResp`: 用户发布物品信息响应
  - `UserStatisticsResp`: 用户统计数据响应
  - `UpdateProfileRequest`: 更新用户资料请求参数
  - `UserProfileResponse`: 用户资料响应

### 5. 数据访问层 (UserMapper, ItemMapper, RoleMapper, PermissionMapper 等)

- **位置**: `/backend/src/main/java/com/campus/lostfound/mapper/`
- **职责**: 数据库操作接口
- **技术**: 使用 MyBatis Plus 框架
- **主要方法**: 基于通用 Mapper 的增删改查操作

### 6. 实体层 (User, Item, Role, UserRole, Permission 等)

- **位置**: `/backend/src/main/java/com/campus/lostfound/domain/entity/`
- **职责**: 对应数据库表的实体类
- **映射**: 与 MySQL 数据库中的 users、items、roles、user_roles、permissions 等表对应

### 7. 工具类

- **SecurityUserUtils**: 用于获取当前用户身份信息
- **LambdaQueryWrapper**: MyBatis Plus 提供的查询构造器
- **BeanUtils**: Spring 框架提供的对象属性复制工具
