------

### 各层详细说明

### 示例 ： 举报模块

#### 1. Controller 层 (ReportController)

**文件位置**: `report/ReportController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |----|----|----| |GET|/api/reports|分页查询举报列表，支持状态和目标类型| |GET|/api/reports/{id}|获取举报详情| |POST|/api/reports|创建举报| |PUT|/api/reports/{id}|更新举报| |DELETE|/api/reports/{id}|删除举报|

------

#### 2. Service 接口层 (ReportService)

**文件位置**: `report/service/ReportService.java`

定义了举报模块的业务接口：

```
public interface ReportService {
    Page&lt;ReportResp&gt; getReportsPage(Integer pageNum, Integer pageSize, String status, String targetType);
    ReportResp getReportById(Long id);
    void createReport(CreateReportReq request);
    boolean updateReport(UpdateReportReq request);
    boolean deleteReport(Long id);
}
```

------

#### 3. DTO 层

数据传输对象：

| 类 | 用途 | |----|----| |CreateReportReq|创建举报请求: targetType, targetId, reason, detail| |UpdateReportReq|更新举报请求: id, targetType, targetId, reason, detail, status, handleRemark| |ReportResp|举报响应: id, bizId, reporterId, targetType, targetId, reason, detail, status, handleRemark, handledAt, createTime|

------

#### 4. Mapper 层 (ReportMapper)

**文件位置**: `mapper/ReportMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

- selectPage() - 分页查询
- selectById() - 按ID查询
- insert() - 插入
- updateById() - 按ID更新
- deleteById() - 按ID删除

------

#### 5. 实体层 (Report)

数据库表对应的实体类，字段包括：
- id (主键)
- biz_id (业务ID)
- reporter_id (举报人ID)
- target_type (目标类型: ITEM/USER等)
- target_id (目标ID)
- reason (举报原因)
- detail (详情)
- status (状态: PENDING/RESOLVED/DISMISSED)
- handle_remark (处理备注)
- handled_at (处理时间)
- handled_by (处理人ID)
- create_time (创建时间)
