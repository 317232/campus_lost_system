## 背景

当前系统支持失物和招领分别发布，但用户需要手动查找匹配项。例如：用户丢失"黑色双肩包"，需要自己去招领列表搜索"书包"来碰运气。

**目标**：当用户查看某条失物（招领）详情时，自动推荐可能匹配的招领（失物）。

---

## 功能需求

### 核心逻辑

当用户查看失物详情时，根据以下维度匹配招领信息：

| 匹配维度 | 权重 | 说明 |
|----------|------|------|
| 物品名称相似度 | 高 | 如"钱包"匹配"钱包"、"卡包" |
| 分类相同 | 高 | 同一分类（电子产品/书本/衣物等） |
| 地点相近 | 中 | 同区域（如都在"教学楼"） |
| 时间相近 | 低 | 7天内 |

### 推荐位置

- **失物详情页** → 显示"可能匹配的招领信息"模块
- **招领详情页** → 显示"可能匹配的失物信息"模块

### 推荐数量

每页显示 2-3 条，按匹配度排序。

## 技术实现

### 后端实现

**新增接口**：`GET /api/items/{id}/matches`

**请求参数**：
```bash
curl "http://localhost:8080/api/items/1/matches?limit=3"
```

**响应格式**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "scene": "found",
    "items": [
      {
        "id": 5,
        "title": "教学楼捡到黑色钱包",
        "itemName": "钱包",
        "category": "电子产品",
        "zone": "教学楼",
        "location": "A栋101",
        "timeLabel": "2026-04-24",
        "thumbnail": "http://...",
        "matchScore": 85
      }
    ]
  }
}
```

**匹配算法**（简单版本）：
1. 根据当前物品的 `scene` 确定查找方向（lost→found 或 found→lost）
2. 查询同 `category` 的物品
3. 计算标题/物品名的关键词重叠度
4. 按匹配分排序返回

**代码位置**：
- `ItemController.java` — 新增 `/api/items/{id}/matches` 接口
- `ItemService.java` — 新增 `getMatches(Long itemId, Integer limit)` 方法
- `ItemServiceImpl.java` — 实现匹配算法

### 前端实现

**新增组件**：`MatchRecommendations.vue`

## 参考代码

### 后端伪代码

```java
public MatchResp getMatches(Long itemId, Integer limit) {
    Item item = itemMapper.selectById(itemId);
    String oppositeScene = item.getScene().equals("lost") ? "found" : "lost";

    // 查询同分类物品
    List<Item> candidates = itemMapper.selectList(
        new LambdaQueryWrapper<Item>()
            .eq(Item::getScene, oppositeScene)
            .eq(Item::getCategory, item.getCategory())
            .eq(Item::getStatus, "PUBLISHED")
    );

    // 计算匹配分
    List<MatchItem> matches = candidates.stream()
        .map(c -> {
            int score = calculateMatchScore(item, c);
            return new MatchItem(c, score);
        })
        .sorted(Comparator.comparing(MatchItem::getScore).reversed())
        .limit(limit)
        .collect(Collectors.toList());

    return new MatchResp(oppositeScene, matches);
}

int calculateMatchScore(Item source, Item target) {
    int score = 0;
    // 分类相同 +30
    if (source.getCategory().equals(target.getCategory())) score += 30;
    // 地点相同 +25
    if (source.getZone().equals(target.getZone())) score += 25;
    // 名称包含 +25
    if (target.getItemName().contains(source.getItemName()) ||
        source.getItemName().contains(target.getItemName())) score += 25;
    // 时间7天内 +20
    // ...
    return score;
}
```
