# 失物招领模块架构图

## 架构概览

```
前端 → LostItemController → LostItemService → LostItemServiceImpl → 数据访问层(MyBatis Plus) → MySQL
                                      ↓
                                   DTO层(LostDTO)
```

## 各层详细说明

### 1. Controller 层 (LostItemController)

- **位置**: `/backend/src/main/java/com/campus/lostfound/lost/LostItemController.java`
- **职责**: 处理 HTTP 请求，提供 RESTful API 接口
- **主要接口**:
  - `GET /api/lost-items`: 分页查询失物信息列表，支持关键词搜索
  - `GET /api/lost-items/{id}`: 获取特定失物信息详情
  - `POST /api/lost-items`: 创建新的失物信息
  - `PUT /api/lost-items/{id}`: 更新失物信息
  - `DELETE /api/lost-items/{id}`: 删除失物信息
- **异常处理**: 包含参数验证和业务异常处理
- **返回格式**: 统一使用 ApiResponse 格式

### 2. Service 接口层 (LostItemService)

- **位置**: `/backend/src/main/java/com/campus/lostfound/lost/service/LostItemService.java`
- **职责**: 定义失物招领业务逻辑接口
- **主要方法**:
  - `getLostItemsPage`: 分页获取失物信息
  - `getLostItemById`: 根据 ID 获取失物详情
  - `createLostItem`: 创建失物信息
  - `updateLostItem`: 更新失物信息
  - `deleteLostItem`: 删除失物信息

### 3. Service 实现层 (LostItemServiceImpl)

- **位置**: `/backend/src/main/java/com/campus/lostfound/lost/service/impl/LostItemServiceImpl.java`
- **职责**: 实现具体的业务逻辑
- **核心逻辑**:
  - 使用 MyBatis Plus 进行数据库操作
  - 实现分页查询功能
  - 数据实体与 DTO 之间的转换
  - 参数验证
  - 设置默认值（如 scene="lost", stage="active"等）
- **依赖注入**: 注入 ItemMapper 进行数据访问

### 4. DTO 层 (LostDTO)

- **位置**: `/backend/src/main/java/com/campus/lostfound/lost/dto/LostDTO.java`
- **职责**: 数据传输对象，用于在不同层级间传递数据
- **主要类型**:
  - `LostItemResp`: 响应给前端的失物信息
  - `CreateLostReq`: 创建失物请求参数
  - `UpdateLostReq`: 更新失物请求参数

### 5. 数据访问层 (ItemMapper)

- **位置**: `/backend/src/main/java/com/campus/lostfound/mapper/ItemMapper.java`
- **职责**: 数据库操作接口
- **技术**: 使用 MyBatis Plus 框架
- **主要方法**: 基于通用 Mapper 的增删改查操作

### 6. 实体层 (Item)

- **位置**: `/backend/src/main/java/com/campus/lostfound/domain/entity/Item.java`
- **职责**: 对应数据库表的实体类
- **映射**: 与 MySQL 数据库中的 items 表对应

### 7. 工具类

- **SecurityUserUtils**: 用于获取当前用户身份信息
- **BeanUtils**: Spring 框架提供的对象属性复制工具
