# 认领申请模块架构图

## 架构概览

```
前端 → ClaimController → ClaimService → ClaimServiceImpl → 数据访问层(MyBatis Plus) → MySQL
                                    ↓
                                 DTO层(ClaimDTO)
```

## 各层详细说明

### 1. Controller 层 (ClaimController)

- **位置**: `/backend/src/main/java/com/campus/lostfound/claim/ClaimController.java`
- **职责**: 处理 HTTP 请求，提供 RESTful API 接口
- **主要接口**:
  - `POST /api/claims`: 提交认领申请
  - `GET /api/users/me/claims`: 获取当前用户的认领记录
- **异常处理**: 包含参数验证和业务异常处理
- **返回格式**: 统一使用 ApiResponse 格式

### 2. Service 接口层 (ClaimService)

- **位置**: `/backend/src/main/java/com/campus/lostfound/claim/service/ClaimService.java`
- **职责**: 定义认领业务逻辑接口
- **主要方法**:
  - `createClaim`: 创建认领申请
  - `getMyClaims`: 获取当前用户的认领记录

### 3. Service 实现层 (ClaimServiceImpl)

- **位置**: `/backend/src/main/java/com/campus/lostfound/claim/service/impl/ClaimServiceImpl.java`
- **职责**: 实现具体的业务逻辑
- **核心逻辑**:
  - 验证用户是否已登录
  - 验证物品是否存在且为拾物信息
  - 防止用户认领自己发布的物品
  - 防止重复认领同一物品
  - 生成认领业务 ID
  - 设置认领状态为"REVIEW_PENDING"
  - 将认领信息保存到数据库
  - 返回包含物品信息的认领记录
- **依赖注入**: 注入 ClaimMapper、ItemMapper、UserMapper 进行数据访问和 SecurityUserUtils 获取当前用户

### 4. DTO 层 (ClaimDTO)

- **位置**: `/backend/src/main/java/com/campus/lostfound/claim/dto/ClaimDTO.java`
- **职责**: 数据传输对象，用于在不同层级间传递数据
- **主要类型**:
  - `CreateClaimReq`: 创建认领请求参数
  - `ClaimResp`: 认领记录响应数据

### 5. 数据访问层 (ClaimMapper, ItemMapper, UserMapper)

- **位置**: `/backend/src/main/java/com/campus/lostfound/mapper/`
- **职责**: 数据库操作接口
- **技术**: 使用 MyBatis Plus 框架
- **主要方法**: 基于通用 Mapper 的增删改查操作

### 6. 实体层 (Claim, Item, User)

- **位置**: `/backend/src/main/java/com/campus/lostfound/domain/entity/`
- **职责**: 对应数据库表的实体类
- **映射**: 与 MySQL 数据库中的 claims、items、users 表对应

### 7. 工具类

- **SecurityUserUtils**: 用于获取当前用户身份信息
- **LambdaQueryWrapper**: MyBatis Plus 提供的查询构造器
