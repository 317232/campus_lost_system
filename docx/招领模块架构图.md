### 各层详细说明

### 示例 ： 招领模块

#### 1. Controller 层 (FoundItemController)

**文件位置**: `found/FoundItemController.java`

- 负责接收 HTTP 请求
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |------|------|------| | GET | `/api/found-items` | 分页查询招领列表，支持关键词搜索 | | GET | `/api/found-items/{id}` | 获取指定招领详情 | | POST | `/api/found-items` | 创建新的招领信息 | | PUT | `/api/found-items/{id}` | 更新招领信息 | | DELETE | `/api/found-items/{id}` | 删除招领信息 |

------

#### 2. Service 接口层 (FoundItemService)

**文件位置**: `found/service/FoundItemService.java`

定义了招领模块的业务接口，封装业务流程：

```
public interface FoundItemService {
    Page<FoundItem> getFoundItemsPage(Integer pageNum, Integer pageSize, String keyword);
    FoundItem getFoundItemById(Long id);
    boolean createFoundItem(FoundItem foundItemDto);
    boolean updateFoundItem(FoundItem foundItemDto);
    boolean deleteFoundItem(Long id);
}
```

------

#### 3. Service 实现层 (FoundItemServiceImpl)

**文件位置**: `found/service/Impl/FoundItemServiceImpl.java`

实现具体的业务逻辑：

| 业务方法 | 核心逻辑 | |---------|---------| | **getFoundItemsPage()** | ① 创建分页对象 ② 构建查询条件，scene="found" ③ 关键词搜索 itemName/description ④ 分页查询 ⑤ 实体转换 DTO | | **getFoundItemById()** | ① 按ID查询 ② 验证 scene="found" ③ 转换 DTO 返回 | | **createFoundItem()** | ① 验证必填字段(itemName, description, location) ② 获取当前用户ID ③ 生成bizId(FND+UUID) ④ 设置scene="found", stage="active", auditStatus="PENDING" ⑤ 插入数据库 | | **updateFoundItem()** | ① 验证ID ② 查询并验证 scene="found" ③ 属性拷贝(排除不可改字段) ④ 更新数据库 | | **deleteFoundItem()** | ① 查询并验证 scene="found" ② 按ID删除 |

------

#### 4. DTO 层 (FoundItem)

**文件位置**: `found/dto/FoundItem.java`

数据传输对象，用于请求和响应：

| 字段 | 用途 | |------|------| | id | 物品ID | | itemName | 物品名称 | | description | 物品描述 | | location | 拾取地点 | | time | 拾取时间 | | contactName | 联系人姓名 | | status | 状态 | | userId | 发布者ID | | createTime | 创建时间 |

------

#### 5. Mapper 层 (ItemMapper)

**文件位置**: `mapper/ItemMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

- `selectPage()` - 分页查询
- `selectById()` - 按ID查询单条记录
- `insert()` - 插入记录
- `updateById()` - 按ID更新
- `deleteById()` - 按ID删除

------

#### 6. 实体层 (Item)

**文件位置**: `domain/entity/Item.java`

数据库表对应的实体类：

| 字段 | 说明 | |------|------| | id | 主键ID | | bizId | 业务ID(FND+UUID) | | scene | 场景(lost/found) | | stage | 阶段 | | status | 状态 | | auditStatus | 审核状态 | | itemName | 物品名称 | | description | 物品描述 | | location | 地点 | | timeLabel | 时间标签 | | contactVisibility | 联系人可见性 | | ownerId | 发布者ID | | title | 标题 | | createdAt | 创建时间 |
