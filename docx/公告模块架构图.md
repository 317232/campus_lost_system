### 各层详细说明

### 示例 ： 公告模块

#### 1. Controller 层 (NoticeController)

**文件位置**: `notice/NoticeController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |----|----|----| |GET|/api/notices|分页查询公告列表，支持关键词和发布状态| |GET|/api/notices/{id}|获取公告详情| |POST|/api/notices|创建公告| |PUT|/api/notices/{id}|更新公告| |DELETE|/api/notices/{id}|删除公告|

------

#### 2. Service 接口层 (NoticeService)

**文件位置**: `notice/service/NoticeService.java`

定义了公告模块的业务接口：

```
public interface NoticeService {
    Page&lt;NoticeResp&gt; getNoticesPage(Integer pageNum, Integer pageSize, String keyword, Boolean published);
    NoticeResp getNoticeById(Long id);
    void createNotice(CreateNoticeReq request);
    boolean updateNotice(UpdateNoticeReq request);
    boolean deleteNotice(Long id);
}
```

------

#### 3. DTO 层

数据传输对象：

| 类 | 用途 | |----|----| |CreateNoticeReq|创建公告请求: title, content, published| |UpdateNoticeReq|更新公告请求: id, title, content, published| |NoticeResp|公告响应: id, title, content, published, publishedAt, createTime|

------

#### 4. Mapper 层 (NoticeMapper)

**文件位置**: `mapper/NoticeMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

- selectPage() - 分页查询
- selectById() - 按ID查询
- insert() - 插入
- updateById() - 按ID更新
- deleteById() - 按ID删除

------

#### 5. 实体层 (Notice)

数据库表对应的实体类，字段包括：
- id (主键)
- title (标题)
- content (内容)
- published (是否发布)
- published_at (发布时间)
- create_time (创建时间)
