### 各层详细说明

### 示例 ： 用户模块

#### 1. Controller 层 (ProfileController)

**文件位置**: `user/ProfileController.java`

- 负责接收 HTTP 请求
- 使用 `@Valid` 注解进行参数校验
- 调用 Service 层处理业务逻辑
- 返回统一的 `ApiResponse` 响应格式

| 方法 | 路径 | 功能 | |----|----|----| |GET|/api/users/me|获取当前登录用户资料| |PUT|/api/users/me|更新当前登录用户资料| |GET|/api/users/me/lost-items|获取当前用户发布的失物列表| |GET|/api/users/me/found-items|获取当前用户发布的招领列表| |GET|/api/users/statistics|获取全量用户统计(管理员)| |GET|/api/users/{userId}/lost-items|获取指定用户的失物列表| |GET|/api/users/{userId}/found-items|获取指定用户的招领列表|

------

#### 2. Service 接口层 (ProfileService)

**文件位置**: `user/service/ProfileService.java`

定义了用户模块的业务接口：

```
public interface ProfileService {
    UserResp getCurrentUserProfile();
    UserProfileResponse updateCurrentProfile(UpdateProfileRequest request);
    List&lt;UserItemResp&gt; getUserLostItems(Long userId);
    List&lt;UserItemResp&gt; getUserFoundItems(Long userId);
    List&lt;UserStatisticsResp&gt; getAdminStatistics();
}
```

------

#### 3. DTO 层

数据传输对象：

| 类 | 用途 | |----|----| |UpdateProfileRequest|更新资料请求: name, phone, email, major, avatarUrl| |UserResp|用户响应: id, studentNo, name, phone, email, major, avatarUrl, status| |UserProfileResponse|更新资料响应| |UserItemResp|用户物品响应: item相关信息| |UserStatisticsResp|用户统计响应|

------

#### 4. Mapper 层

**文件位置**: `mapper/UserMapper.java`, `mapper/ItemMapper.java`

继承 MyBatis-Plus 的 `BaseMapper`，提供数据库操作：

| Mapper | 操作 | |----|----| |UserMapper|用户表CRUD| |ItemMapper|物品表查询，通过scene和owner_id筛选|

------

#### 5. 实体层 (User)

数据库表对应的实体类，字段包括：
- id (主键)
- student_no (学号，唯一)
- name (姓名)
- password_hash (加密密码)
- phone (电话)
- email (邮箱)
- major (专业)
- avatar_url (头像)
- status (状态: ACTIVE/INACTIVE)
- create_time (创建时间)

------

#### 6. 工具类

| 工具类 | 位置 | 功能 | |--------|------|------| |**SecurityUserUtils**|security/SecurityUserUtils.java|获取当前登录用户ID，未登录时fallback到1L|
